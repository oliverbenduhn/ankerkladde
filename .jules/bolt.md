## 2024-05-15 - [getDatabase Overhead Optimization]
**Learning:** Found that `getDatabase()` (which handles schema initialization) was unconditionally running several expensive legacy data migration loops (`migrateLegacyCategories`, `backfillLegacyCategoryKeys`, etc) and an icon-update loop on *every request*. This caused massive N+1 overhead during the connection phase.
**Action:** Always check the connection bootstrapping functions (`getDatabase()`) to ensure heavy migration scripts are guarded behind a one-time execution flag (e.g., `hasDatabaseMetaFlag`), rather than running unconditionally on every request.
## 2024-06-25 - [getDatabase Default Category Initialization N+1 Bug]
**Learning:** Found that `ensureDefaultCategories($db)` inside `db.php` was being called on every request because it lacked the `hasDatabaseMetaFlag` wrapper. As the application gained more users, iterating over all users to check and potentially re-insert default categories turned into a severe O(N) performance bottleneck for `getDatabase()` initialization.
**Action:** Always wrap heavy, iterative "ensure" loops in the database initialization pipeline in a one-time migration flag (`hasDatabaseMetaFlag`). Handle the creation of specific user assets (like default categories) explicitly when the user is created (e.g. `public/admin.php` and `scripts/create-user.php`), rather than continually sweeping the database on each request.
